#### (3) Dynamic Routing
- /blog/1 , /blog/2와 같은 ID의 경우에는 다이나믹라우팅을 사용한다.
- /blog/[id] 로 구분하며, params를 받아서 해결한다.

```ad-note
title: Props의 값

params는 {id : 3433}
searchParams는 {region:kr}  -> 즉 쿼리값

```

#### (5) Not Found

각 폴더 아래 **not-found.tsx** 파일을 만들면 자동으로 라우팅되어짐 
- 자신의 폴더에 not-found.tsx가 없다면 상위폴더로 not-found를 찾아서 보여준다.,


```typescript
// page.tsx
// 1000번이 넘어가면, Not-Found를 발생
export default function ReviewDetail({
  params,
}: {
  params: { productId: string; reviewId: string };
}) {
  console.log(params);
  if(parseInt(params.reviewId)>100){
    notFound();
  }
  return <div>ReviewDetail {params.reviewId}</div>;
}

// not-found.tsx
export default function ReviewNotFound(){
    return (
        <div>Review Not Found</div>
    )
}


```
