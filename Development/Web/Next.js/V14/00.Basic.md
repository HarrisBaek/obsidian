title: Library

title: Framework

## 3. SSR vs CSR

- 렌더링이란 우리가 짠 파일을 브라우저가 읽을수 있는 **HTML**로 변환하는 작업이다.

```ad-done
title: CSR - Client Side Rendering
클라이언트 단에서 HTML를 만드는 작업을 수행하는것

`단점`
- USER가 첫 접속시 자바스크립트가 작동해서 렌더링 될때까지 시간이 걸린다.
- 검색엔진이 최적화 되지 않는다. (렌더링이 일어나지 않았으니깐)
```

```ad-done
title: SSR - Server Side Rendering
서버에서 요청시 HTML를 만드는 작업을 수행하는것

`단점`
- USER가 첫 접속시 자바스크립트가 작동해서 렌더링 될때까지 시간이 걸린다.
- 검색엔진이 최적화 되지 않는다. (렌더링이 일어나지 않았으니깐)
```

## 4. Hydration

```ad-summary
title:하이드레이션 (Hydration) 이란?
  
서버사이드 렌더링(SSR)을 통해 만들어 진 인터랙티브 하지 않는 HTML을 클라이언트 측 자바스크립트를 사용하여 인터랙티브한 리액트 컴포넌트로 변환하는 과정을 말한다.  
(서버 환경에서 이미 렌더링된 HTML에 React를 붙이는 것)
```

(Ex. Link 태그를 쓰면, 서버에서 Anchor 태그로 전달해줘서, 페이지 이동시마다 딸깍 하지만, 이후 Hydration되면, Link태그로 작동하여 캐싱되어 한번 로드한건 로드하지 않게 된다.)

### "Use Client"
	 Use Client의 뜻은 그 컴포넌트가 클라이언트에서 **모두** 렌더링 된다는 뜻이 아니라, 
	 서버에서 렌더링이 된 이후 User-Interaction 부분만 클라이언트에서 렌덩이 한다는 뜻이다. (hydration)
