
#### (1) Private Folder
 - 라우팅 시스템에 고려되지 않아야 하는 별도의 개인 폴더 (컴포넌트, 라이브러리, UI등)
 - `언더바(_lib) 로 시작하는 폴더의 경우 라우팅으로 인식하지 않는다.

#### (2) Routing Group
- 앱 디렉토리에서는 중첩된 폴더가 일반적으로 URL 경로에 매핑됩니다. 그러나 폴더를 Route 그룹으로 표시하여 해당 폴더가 경로의 URL 경로에 포함되지 않도록 할 수 있습니다.

- 이를 통해 URL 경로 구조에 영향을주지 않고 라우트 세그먼트와 프로젝트 파일을 논리적으로 구성할 수 있습니다.

- 사이트 섹션, 의도 또는 팀별로 경로를 구성 할 때
- 동일한 라우트 세그먼트 수준에서 중첩된 레이아웃을 활성화 할 때

- 여러 루트 레이아웃을 포함하여 동일한 세그먼트에 여러 중첩 레이아웃 만들기
- 특정 세그먼트를 레이아웃으로 선택 할 때


#### (3) Parallel Routes

- Slot이란 개념으로 Layout.tsx에서 사용가능 
-  동시에 또는 조건에 따라 동일한 레이아웃에서 하나 이상의 페이지를 렌더링할 수 있게 해줍니다.
- 대시보드나 소셜 사이트의 피드와 같은 매우 동적인 앱 섹션에서는 병렬 라우팅을 사용하여 복잡한 라우팅 패턴을 구현할 수 있습니다.

#### (6) Conditional Routes
- 로그인등의 특정 조건들에 의해서 페이지가 달리 표현되어야 할 경우
```typescript
export default function DashboardLayout({
  children,
  users,
  revenue,
  notifications,
  login,
}: {
  children: React.ReactNode;
  users: React.ReactNode;
  revenue: React.ReactNode;
  notifications: React.ReactNode;
  login: React.ReactNode;
}) {
  const isLoggedIn = false;
  
// 로그인 환경에 따라 화면을 다르게 보여준다.
  return isLoggedIn ? (
    <div>
      <div>{children}</div>
      <div style={{ display: "flex" }}>
        <div style={{ display: "flex", flexDirection: "column" }}>
          <div>{users}</div>
          <div>{revenue}</div>
        </div>
        <div style={{ display: "flex", flex: 1 }}>{notifications}</div>
      </div>
    </div>
  ) : (
    login
  );
}

```

#### (7) Intercepting Routes
- 클릭해서 이동시에는 내부적으로 보이지만, 새로 로드시(url 접근시)에는 전체 페이지로 로드된다.

![[스크린샷 2024-02-11 오후 5.29.49.png]]

- Link가 발생하는 같은 폴더 레벨에서 (.) or(..) 처럼 목표로하는 page의 같은 레벨의 Intercepted 폴더를 만들면, 거기서 일어난 Link는 모두 거기로 발생. 

- 예를들어 f1/f4/page.tsx에서 f1/f3로 Link가 발생하면 f1/r4/(..)f3/page.tsx 를 만들면 Link시 거기로 인터셉트가 발생한다.
- (...) 의 경우 Relative Path로 App/ 밑으로 표기된다.
