#### (2) Nested Layout
- 내부의 layout은 각 부모의 내부로 들어가서 겹쳐진다.
- 기본적으로 한 앱에 두~세개의 레이아웃을 가지고 있다.

```typescript

<RootLayout>
	<Nested Layout>
		<Page></Page>
	</Nested Layout>
</RootLayout>
```

![[스크린샷 2024-02-10 오후 10.35.06.png]]

```typescript
export default function ProductDetailsLayout({
  children,
}: {
  children: React.ReactNode;
}){
    return(
        <div>
            {children}
            <h2>Features products</h2>
        </div>
    )
}

```

#### (3) Template
- 기본적으로 Layout은 페이지를 왔다갔다 하여도 리렌더링이 되지 않는다.
- Layout에게도 마운트 되어야 하는 상황이 발생하면, 그때 Template 사용 가능하다.
 (실시간으로 변경되어야하는 Layout 느낌?)
- **Layout과 Template는 공존할수 없다**. (공식문서에는 특별한 이유없으면 Layout을 써라고 가이드)
