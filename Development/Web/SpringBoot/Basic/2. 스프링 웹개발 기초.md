## 제공 가능한 서버 타입 3가지
* 정적 컨텐츠 
* MVC와 템플릿 엔진 
* API


### 정적 컨텐츠
-  단순한 HTML 페이지만을 보여준다. (Thymeleaf등을 사용할 수 없다.)
-  resources/static 에 HTML를 위치 한다.
- URL은 html full name으로 접속한다 
`ex) http://localhost:8080/hello-static.html`

![[static_iamge.png]]
>첫 번째로는 Controller에서 hello-static 관련 컨트롤러가 있는지 찾아본다. 없다면 resource: static/hello-static.html가 있으면 그것을 전달한다.

### MVC와 템플릿 엔진
* MVC: Model, View, Controller


**Controller
```java
@Controller
  public class HelloController {

      @GetMapping("hello-mvc")
      public String helloMvc(@RequestParam("name") String name, Model model) {

          model.addAttribute("name", name);

          return "hello-template";
      }

}
```


**View**
```HTML
//resources/templates/hello-template.html

<html xmlns:th="http://www.thymeleaf.org">
<body>
	 <p th:text="'hello ' + ${name}">hello! empty</p>
 </body>
</html>

```


> http://localhost:8080/hello-mvc?name=Harris

### API 
- `ResponseBody`를 사용하면 ViewResolver를 사용하지 않음
- 대신에 HTTP의 Body에 문자 내용을 직접 반환(HTML Body Tag를 말하는것이 아님)



**@ResponseBody 문자 반환**
``` java
@Controller

  public class HelloController {

      @GetMapping("hello-string")
      @ResponseBody
      public String helloString(@RequestParam("name") String name) {

          return "hello " + name;
      }

}
```

**@ResponseBody 객체 반환**
- `@ResponseBody`를 사용하고, 객체를 반환하면 객체가 Json으로 변환됨

> json이 xml에 비해서 가볍고 간단하다. 그리하여 기본으로 json으로 반환한다

```java
@Controller

  public class HelloController {

      @GetMapping("hello-api")
      @ResponseBody
      public Hello helloApi(@RequestParam("name") String name) {

          Hello hello = new Hello();
          hello.setName(name);
          return hello;
      }      

static class Hello {
          private String name;
          public String getName() { return name; }
          public void setName(String name) {this.name = name;} 
	  }
}
```
